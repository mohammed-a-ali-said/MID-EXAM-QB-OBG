Admin Disabled On Public Site

Use the protected OAuth admin app instead.

This public GitHub Pages site does not expose the editor anymore. Content editing now belongs to the separate protected admin app with GitHub login and server-side save routes.

The old static admin page was intentionally disabled because a public static editor cannot be securely protected on GitHub Pages.
  • Student study access stays on this public site.
  • Content editing now happens in the separate obg-admin-next app.
  • That secure app should run locally or be deployed to a protected host like Vercel.
